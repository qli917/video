# Anti-PPT Rules

## PPT 感识别

出现以下模式即视为风险：一页一个观点、中央悬浮小卡片、连续相同布局、纯文字页、大面积空背景、长时间静止窗口、统一淡入淡出、结尾总结页，或主要靠字幕解释而非画面证明。

### 白底 PPT

白底不是问题，空白底加知识卡才是。用放大的真实页面、截图纹理、手写批注、主体遮挡和连续动作填充视觉信息。

### 深色 PPT

深色背景加发光边框、数据卡片和小字号说明仍是 PPT。把终端、时间线、旧视频或数据页放大为主视觉，让文字直接压在真实素材上。

### 科技 UI PPT

虚构仪表盘、AI 工作台、卡片轮播和加载动画不能代替证据。数据应来自真实表格，分析应来自真实文档，过程应来自真实录屏。

## 替代方法

- 一页一个观点 → 一个真实动作或对比承载观点。
- 小卡片 → 主体占画面 55%—80%，背景继续保留真实内容。
- 静态终端 → 只保留输入、执行、结果三个短动作。
- 静态图表 → 数字冲突、曲线下坠和对应旧片交叉剪辑。
- 总结页 → 让修改后成片继续播放，轻量覆盖 CTA。

## 占比限制

- 真实数据、真实视频、真实录屏和真实对比：至少 70%—80%。
- 装饰性 UI 和 AI 画面：合计不超过 10%。
- 纯文字画面：0%。
- 连续卡片镜头：0；同构卡片不得连续出现。
- 空背景：接近 0%。

抽帧后若三张相邻关键帧具有相同模板布局，重做其中至少两张主视觉。
