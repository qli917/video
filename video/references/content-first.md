# Content First

## Skill 不能当内容

观众关心的是问题、证据、变化和结果，不关心制作方用了多少 Skill。工具列表只能解释制作方式，不能替代观看理由。若第一屏先出现工具名，陌生用户无法判断“这与我有什么关系”。

## 内容证据链

按以下顺序建立内容：

1. **真实输入**：原视频、数据导出、网页、代码、文档、评论或用户问题。
2. **真实资料**：可核查的来源链接、公开案例、原始表格和时间戳。
3. **真实失败**：旧版本、错误结果、留存下坠或操作痛点。
4. **真实修改**：删除、移动、重录、重写、压缩或重新渲染的可见动作。
5. **真实结果**：修改后成片、前后对比、导出文件或新一轮数据。

缺少任何关键环节时，在 `source_package.md` 标记。无法证明问题、过程或结果时，先补素材，不生成虚构仪表盘。

## 合格内容包

- 每个核心结论至少对应一个真实文件或真实画面。
- 每个“修改有效”的说法至少对应一组 Before / After。
- 数据需注明作品、时间范围、口径和计算方式，禁止混用不同作品。
- 涉及账号、手机号、路径、聊天或后台信息时先脱敏。
- 装饰素材不能承担关键事实证明。
